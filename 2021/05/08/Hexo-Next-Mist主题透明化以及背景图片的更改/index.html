Hexo-Next-Mist主题透明化以及背景图片的更改

发表于 2021-05-08 更新于 2021-05-09 分类于 网页搭建 阅读次数：
本文字数： 3.8k 阅读时长 ≈ 3 分钟

1.在 hexo/themes/ next/source/css/ _ custom下无 _custom.styl 文件，自己添加 _custom.styl编辑后，不能实现背景图片自定义；2.找到主题配置文件 _config.yml并打开，按住 ctrl + f搜索 custom_file_path:把style: source/_data/ _styles.styl注释取消掉， 但source文件下无_data/styles.styl文件，自己添加文件编辑后，也不能实现背景图片自定义。两种情况都不能解决时，可看本文。

问题描述

我前期博客的搭建按照大佬教程 个人博客第8篇——优化主题（持续更新） - 知乎 (zhihu.com) 进行。板娘按照教程 终于等到你！不蒜子-统计工具和萌萌哒-2D模型~ | 大大的小屋 (gitee.io) 。

对于hexo-next-mist 主题的背景图片修改，当以下两种情况都不能解决时，当然我都遇到了，呜呜~。

​ 1.在 hexo/themes/ \next\source\css\ _custom下无**_custom.styl** 文件，自己添加**_custom.styl**编辑后，不能实现背景图片自定义。

​ 2.找到主题配置文件 _config.yml 并打开，按住 ctrl + f 搜索 custom_file_path: 把 style: source/_data/ _styles.styl 注释取消掉， 但source文件下无**_data/styles.styl**文件，自己添加文件编辑后，仍不能实现背景图片自定义

问题解决

更改博客背景

打开文件themes/next/source/css/_schemes/Mist/ _layout.styl文件，在末尾添加如下代码

代码解释
  • background:url为图片路径，将图片重命名为background.jpg放在\themes\next\source\images下，当然也可以直接使用链接，图片上传至图床，生成url，（好用免费图床： Image Upload - SM.MS - Simple Free Image Hosting） 。
  • background-repeat：若果背景图片不能全屏，那么是否平铺显示，充满屏幕
  • background-attachment：背景是否随着网页上下滚动而滚动，fixed为固定
  • background-position：就是调整图片的位置
  • background-size：图片展示大小，这里设置100% 100%的意义为：如果背景图片不能全屏，那么是否通过拉伸的方式将背景强制拉伸至全屏显示。
